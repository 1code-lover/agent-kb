## 测试步骤摘要

### 1. 路由层验证

执行：

```python
route_agent_task("hello", "agent")
```

结果：

- 返回 `{"tool_name": "llm_chat", "reason": "explicit_agent_mode"}`

结论：

- `mode=agent` 已明确走模型调用链路，不再指向 `kb_search`

### 2. API 层真实链路验证

步骤：

1. 新增 `Stage3TestProvider`
2. 将当前模型切换到 `Stage3TestProvider / stage3-model-b`
3. 调用 `POST /api/agent/run`，请求体中设置 `mode=agent`

结果：

- 接口返回 `503`
- 该错误来源于远端模型调用失败，而不是知识库结果返回

结论：

- API 层已经真实走到 Agent 模型调用链路
- 当前失败原因是测试 provider 不可用，不是路由错误，也不是回退到知识库

### 3. 运行时成功路径验证

步骤：

1. 通过 `FALLBACK_CONFIG_STORE` 注入当前模型配置
2. 对 `agent_tools._call_openai_compatible` 做可控 mock
3. 直接调用 `api.services.agent_runtime.run_agent()`

结果：

- `answer` 正常返回 mock 内容
- `task_state.status = completed`
- `plan` 中 `plan-model / plan-answer / plan-trace` 全部为 `completed`
- `steps[0].step = llm_chat`
- `steps[0].summary = Stage3MockProvider / stage3-mock-model`
- `timeline` 中包含：
  - 用户问题
  - 模型调用步骤
  - assistant 回答
- `receipts[0].tool_name = llm_chat`
- `receipts[0].input/output` 中都包含当前 `provider/model`
- `pending_actions = []`

结论：

- 成功路径已经满足 Stage 3 对 timeline、receipt、当前模型标识的要求
- 运行过程不依赖知识库 evidence
