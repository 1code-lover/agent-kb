# ThinkRAG 智能体开发助手 V1 功能设计（FRD）

## 1. 文档目的
将 PRD 转换为可执行的功能清单，为研发排期、联调和测试设计提供统一依据。

## 2. 功能模块总览
1. 工作台框架模块
2. 任务生命周期模块
3. 执行时间线模块
4. 风险审批模块
5. 知识证据模块
6. 历史回放模块
7. 系统状态与异常处理模块

## 3. 功能清单（按优先级）

## 3.1 P0（必须）

### M1 工作台框架
- 目标：提供统一任务入口和三栏布局。
- 包含：
  - 左栏：会话列表、当前任务状态。
  - 中栏：任务输入、执行流、控制按钮。
  - 右栏：上下文页签（Evidence/Files/Runtime）。
- 验收要点：
  - 三栏布局稳定可用。
  - 主流程不需跳出工作台。

### M2 任务生命周期管理
- 目标：支持任务从开始到完成的状态转换。
- 包含：
  - 新建任务、继续任务、终止任务。
  - 状态机：idle/planning/running/waiting_approval/failed/completed。
- 验收要点：
  - 状态切换准确。
  - 中断后可恢复到可继续状态。

### M3 执行时间线
- 目标：展示 Agent 执行轨迹。
- 包含：
  - 步骤事件列表（计划、工具调用、结果、异常）。
  - 步骤展开查看摘要信息。
- 验收要点：
  - 至少展示“做了什么、结果如何、当前状态”。
  - 错误步骤可见并有处理建议。

### M4 风险审批（基础版）
- 目标：保障中高风险动作可控。
- 包含：
  - 风险等级映射（L0-L3）。
  - 中高风险动作触发审批。
  - 审批通过/拒绝处理。
- 验收要点：
  - 高风险动作不可静默执行。
  - 审批拒绝后动作停止并反馈替代路径。

## 3.2 P1（重要）

### M5 知识证据关联
- 目标：将知识库作为执行依据层。
- 包含：
  - 执行中知识检索调用。
  - 证据片段在右栏展示并关联步骤。
- 验收要点：
  - 用户能看到“某步骤引用了哪些证据”。

### M6 历史回放
- 目标：支持会话内复盘。
- 包含：
  - 历史执行事件查看。
  - 审批决策记录查看。
- 验收要点：
  - 关键步骤和审批链路可追溯。

## 3.3 P2（增强，可延后）

### M7 失败恢复增强
- 目标：提升执行稳定性和连续性。
- 包含：
  - 从失败步骤重试。
  - 引导用户修正输入后继续执行。

### M8 体验优化
- 目标：提升信息密度和可读性。
- 包含：
  - 筛选仅看异常步骤。
  - 关键状态提示优化。

## 4. 页面级设计（高层）

## 4.1 Agent Workspace（主页面）
- 左栏：`SessionListPanel`
- 中栏：`TaskComposer` + `TimelinePanel`
- 右栏：`EvidencePanel`、`FilePanel`、`RuntimePanel`

## 4.2 Settings（二级页面）
- 模型配置、执行策略、审批默认策略。

## 4.3 Knowledge Base（二级页面）
- 文档管理、索引状态、数据源维护。

## 5. 交互规则
1. 主输入框用于发起和修正任务目标。
2. 执行中允许追加约束，系统按下一轮执行生效。
3. 审批卡片必须阻塞对应高风险步骤。
4. 证据展示默认折叠，避免干扰主流程。

## 6. 异常处理设计
1. 工具执行失败：在步骤卡片显示错误摘要和建议动作。
2. 审批超时：任务进入 paused/waiting_approval 并提示用户处理。
3. 网络中断：前端重连后自动拉取最近事件。

## 7. 开发拆分建议

### 前端
- 工作台框架与状态管理
- 时间线事件渲染
- 审批交互卡片
- 上下文侧栏

### 后端
- 任务编排控制器
- 风险评估与审批处理
- 事件流输出和历史查询
- 知识检索工具接入

### 测试
- 核心场景脚本
- 风险审批路径验证
- 中断恢复与回放验证

## 8. DoD（功能完成定义）
1. P0 模块全部通过测试用例。
2. 至少 3 个真实任务场景演示通过。
3. 审批路径和异常路径可复现可回放。
4. 形成可发布的 V1 功能说明和已知问题列表。
