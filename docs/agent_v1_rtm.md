# ThinkRAG 智能体开发助手 V1 需求追踪矩阵（RTM）

## 1. 说明
- 目的：确保需求、实现、测试、验收一一对应，避免遗漏。
- 规则：每条需求至少对应 1 条测试用例和 1 条验收检查项。

## 2. 需求追踪矩阵

| 需求ID | 需求描述 | 对应功能模块 | 测试类型 | 验收标准 |
| --- | --- | --- | --- | --- |
| FR-01 | 提供三栏工作台主页面 | M1 工作台框架 | 功能测试、UI测试 | 主流程在单页面完成，无强制跳转 |
| FR-02 | 支持任务生命周期状态管理 | M2 任务生命周期 | 功能测试、状态机测试 | 状态切换正确，可暂停/继续/终止 |
| FR-03 | 展示执行步骤流 | M3 执行时间线 | 功能测试、场景测试 | 关键步骤可见，错误步骤可识别 |
| FR-04 | 中高风险动作审批 | M4 风险审批 | 功能测试、安全测试 | L2/L3 动作需审批且不可绕过 |
| FR-05 | 知识库证据引用与展示 | M5 知识证据关联 | 功能测试、场景测试 | 证据来源可见，并关联执行步骤 |
| FR-06 | 支持历史回放与追溯 | M6 历史回放 | 功能测试、回归测试 | 会话内关键步骤和审批记录可回放 |
| NFR-01 | 单页主流程高可用 | M1/M2/M3 | 可用性测试 | 用户可在单界面完成主要操作 |
| NFR-02 | 长任务稳定与恢复 | M2/M3/M6 | 稳定性测试 | 刷新/重连后会话状态可恢复 |
| NFR-03 | 高风险动作可控 | M4 | 安全测试 | 审批覆盖率 100% |
| NFR-04 | 模块可扩展可维护 | 全模块 | 架构评审 | 模块边界清晰，可支持后续扩展 |
| NFR-05 | 执行可追溯 | M3/M6 | 回归测试 | 关键执行事件可查询可回放 |

## 3. 场景测试映射

| 场景ID | 场景描述 | 覆盖需求 |
| --- | --- | --- |
| SC-01 | 输入任务后成功执行并完成 | FR-01, FR-02, FR-03 |
| SC-02 | 执行中出现错误并给出恢复路径 | FR-03, NFR-02 |
| SC-03 | 高风险动作触发审批并通过后继续 | FR-04, NFR-03 |
| SC-04 | 高风险动作审批拒绝并停止危险步骤 | FR-04, NFR-03 |
| SC-05 | 任务步骤引用知识证据并可查看来源 | FR-05 |
| SC-06 | 任务完成后回看全部关键步骤和审批记录 | FR-06, NFR-05 |

## 4. 发布门禁（由 RTM 驱动）
1. FR-01 ~ FR-06 覆盖率 100%。
2. 核心场景 SC-01 ~ SC-04 通过率 100%。
3. 回归场景 SC-05 ~ SC-06 通过率 >= 95%。
4. 阻断问题（安全、崩溃、数据错乱）为 0。
