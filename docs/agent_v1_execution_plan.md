# ThinkRAG 智能体开发助手 V1 实施计划（开发前版）

## 1. 计划目标
在 3 个阶段内完成 V1 的可用、可控、可增强闭环，确保每个阶段都有可演示产物和可验证结果。

## 2. 阶段计划

## 阶段 A：可用闭环（建议第 1-3 周）
### 目标
- 跑通主工作台和单任务执行闭环。

### 关键任务
1. 完成三栏工作台基础布局。
2. 打通任务发起和执行状态展示。
3. 打通执行步骤时间线展示。
4. 完成阶段冒烟测试。

### 阶段产出
- 可演示的端到端流程（单任务）。
- 阶段问题清单和修复优先级。

## 阶段 B：可控执行（建议第 4-5 周）
### 目标
- 建立风险分级和审批闭环，保证中高风险操作可控。

### 关键任务
1. 风险等级映射规则落地。
2. 审批卡片与审批动作联调。
3. 审批拒绝后的恢复/终止路径。
4. 风险相关回归测试。

### 阶段产出
- 风险审批演示脚本。
- 审批流程验证记录。

## 阶段 C：知识增强（建议第 6-7 周）
### 目标
- 将知识库作为执行证据层接入并可视化展示。

### 关键任务
1. 知识证据与执行步骤关联。
2. 右侧证据面板展示优化。
3. 关键场景效果评估。
4. 全量回归与发布评审准备。

### 阶段产出
- 证据关联演示。
- 发布评审材料（测试报告、已知问题、上线建议）。

## 3. 角色与责任
- 产品：范围控制、里程碑评审、变更决策。
- 前端：工作台交互、时间线、审批和上下文展示。
- 后端：编排、风险评估、审批处理、日志能力。
- 测试：测试计划、场景验证、回归把关。

## 4. 风险管理机制
1. 每周至少 2 次同步会（进度 + 风险）。
2. 新增需求必须进入变更评审。
3. 每阶段结束前必须完成质量门禁检查。

## 5. 阶段质量门禁
- 阻断问题（崩溃/风险误执行）= 0。
- 核心场景通过率 >= 95%。
- 审批覆盖率 = 100%（L2/L3）。

## 6. 开发启动建议
1. 启动会当天完成清单勾选（见 `agent_v1_dev_readiness_checklist.md`）。
2. 先锁定阶段 A 的任务和 Owner，再正式开工。
3. 阶段 A 完成后再解锁阶段 B 任务，避免并行失控。
