# ThinkRAG 智能体开发助手 V1 开发前准备检查清单

## 1. 使用说明
- 目的：确保需求、方案、资源、流程在开发前就绪。
- 使用方式：启动会逐项勾选，未完成项必须明确 Owner 和截止时间。

## 2. 需求与方案准备
- [x] PRD 已完成并评审（`docs/agent_v1_prd.md`）
- [x] FRD 已完成并评审（`docs/agent_v1_frd.md`）
- [x] RTM 已完成并评审（`docs/agent_v1_rtm.md`）
- [ ] V1 需求冻结（冻结后新增需求走变更流程）
- [ ] In Scope / Out of Scope 团队达成一致

## 3. 研发准备
- [ ] 前端负责人已确认页面与组件拆分
- [ ] 后端负责人已确认编排与审批实现路径
- [ ] 测试负责人已确认测试策略和关键场景
- [ ] 里程碑排期已确认（阶段 A/B/C）
- [ ] 风险清单已评审并分配 Owner

## 4. 环境与工具准备
- [ ] 本地开发环境可用（Python、Node、依赖安装通过）
- [ ] Web/Desktop 联调流程可用
- [ ] 日志与调试手段可用（前后端）
- [ ] 关键启动脚本可运行（`desktop-dev.ps1` 或等价流程）

## 5. 质量与测试准备
- [ ] 冒烟测试清单已建立
- [ ] 核心场景测试脚本（SC-01~SC-06）已准备
- [ ] 回归测试执行节奏已确定（每阶段里程碑前）
- [ ] 发布门禁标准已确认

## 6. 评审与协作机制
- [ ] 固定评审节奏已建立（建议每周 2 次）
- [ ] 需求变更流程明确（谁提、谁批、何时生效）
- [ ] 里程碑评审人和决策人已明确
- [ ] 文档同步责任人已明确

## 7. 开工准入条件（Go/No-Go）
以下条件全部满足才进入开发：
1. 需求冻结完成。
2. 里程碑排期确认。
3. Owner 明确且风险有应对方案。
4. 冒烟测试与核心场景清单可执行。
