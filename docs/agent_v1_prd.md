# ThinkRAG 智能体开发助手 V1 需求文档（PRD）

## 1. 文档信息
- 文档版本：v1.0
- 文档状态：评审稿
- 适用范围：ThinkRAG Web/Desktop 一体化智能体工作台
- 目标读者：产品、研发、测试、项目管理

## 2. 项目背景
当前 ThinkRAG 的核心价值集中在知识库问答，用户心智是“问答系统”。本项目目标是将产品升级为“Agent-first 开发助手”，以任务执行闭环作为主体验，知识库作为证据增强层。

## 3. 目标与成功标准

### 3.1 业务目标
1. 将主流程从“问答”升级为“开发任务执行”。
2. 建立可观察、可控制、可追溯的智能体执行体验。
3. 将知识库从独立入口能力升级为执行辅助能力。

### 3.2 成功标准（业务层）
1. 用户可在单一工作台完成一次真实开发任务闭环。
2. 用户可查看关键执行步骤并理解执行依据。
3. 中高风险动作无法绕过审批。
4. 知识库证据被智能体实际引用并可回看来源。

## 4. 用户与场景

### 4.1 目标用户
- 个人开发者：关注提效与操作简化。
- 小团队研发负责人：关注可控、可审计和复盘能力。

### 4.2 核心场景
1. 用户输入任务目标（例如：新增缓存并补测试）。
2. 智能体分解步骤并执行（读代码、修改、运行命令、验证）。
3. 高风险动作触发审批，用户决策后继续。
4. 任务结束后输出结果总结、变更说明、风险提示。

## 5. 范围定义

### 5.1 In Scope（V1）
- 单智能体执行闭环。
- 三栏工作台（会话、执行流、上下文）。
- 执行步骤可视化。
- 风险分级和审批流程（基础版）。
- 知识库证据引用展示（基础版）。
- 会话级历史回放。

### 5.2 Out of Scope（V1 不做）
- 多智能体协作。
- 插件生态。
- 企业级多租户权限体系。
- 自动化调度/定时编排。

## 6. 功能需求

### FR-01 工作台主界面
- 提供统一入口承载任务执行。
- 左栏：会话/任务导航。
- 中栏：Agent 对话与执行时间线。
- 右栏：上下文面板（文件、运行态、证据）。

### FR-02 任务生命周期
- 支持创建、继续、暂停、终止任务。
- 支持执行中追加用户指令。
- 任务状态包含：idle、planning、running、waiting_approval、failed、completed。

### FR-03 执行可观察
- 展示每个关键步骤的动作类型、执行状态、结果摘要。
- 失败步骤必须显示错误信息和下一步建议。

### FR-04 风险控制与审批
- 动作风险分级：L0/L1/L2/L3。
- L2/L3 默认触发审批流程。
- 审批拒绝后不得继续执行对应危险动作。

### FR-05 知识库增强
- 智能体可在执行中检索知识库。
- 右侧可展示证据片段、来源和关联步骤。

### FR-06 历史与追溯
- 支持会话内执行步骤回放。
- 支持审批记录与失败链路追溯。

## 7. 非功能需求

### NFR-01 可用性
- 核心流程在一个页面完成。
- 用户可快速识别当前执行状态。

### NFR-02 稳定性
- 长任务执行状态不混乱。
- 页面刷新或短暂断连后可恢复会话状态。

### NFR-03 安全性
- 中高风险动作必须可审批。
- 高危动作支持硬阻断策略（规则化）。

### NFR-04 可维护性
- 模块分层清晰，支持后续扩展多智能体能力。

## 8. 发布分期

### 阶段 A：可用闭环（2-3 周）
- 完成三栏工作台和单任务闭环演示。

### 阶段 B：可控执行（2 周）
- 完成风险分级、审批、拒绝后恢复流程。

### 阶段 C：知识增强（2 周）
- 完成证据关联展示和效果验证。

## 9. 依赖与风险

### 9.1 关键依赖
- FastAPI Agent 接口稳定。
- 前端状态管理和事件流渲染能力。
- 知识库检索质量可用。

### 9.2 关键风险
- 范围膨胀导致延期。
- 智能体执行不稳定导致体验下降。
- 审批链路不完整造成安全风险。

## 10. 验收口径（需求侧）
1. 单页面完成任务闭环。
2. 执行步骤可见、可理解、可回放。
3. 中高风险动作审批覆盖率 100%。
4. 知识库证据可被执行步骤引用并追溯来源。
